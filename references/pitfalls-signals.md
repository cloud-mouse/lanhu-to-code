# pitfalls 识别信号速查（轻量）

> 先读本表定位「失败 N」，再打开 `pitfalls.md` 对应小节展开。交付前仍须完成 `SKILL.md §4.3` 的 24 项核对（可逐条展开本文）。

| N | 快速信号（命中则读 `pitfalls.md` 失败 N） |
|---|------------------------------------------|
| 1 | 代码里大量裸 hex / 裸 px，**且**项目里已有可 1:1 映射的 token 却未使用（无匹配 token 时保留精确值合法） |
| 2 | 手写 `<input>` / `<button>`，项目里已有封装组件 |
| 3 | 出现无文档的 `/api/...` 路径 |
| 4 | 渐变/阴影/边框被收成纯色或简化 |
| 5 | `// ...`、样式同上、大块省略 |
| 6 | 无安全区、无状态栏/底部 inset |
| 7 | 蓝湖 CDN、或错误相对路径 |
| 8 | 只有默认态，无 hover/active/disabled |
| 9 | 组件 prop 名与文档不一致 |
| 10 | rpx/rem/px 与 `UNIT_STRATEGY` 不一致 |
| 11 | 设计稿没有的装饰、标题条、分割线 |
| 12 | 设计有、代码无；或区域漏节点 |
| 13 | 整页默认白底，设计有背景 |
| 14 | 同区域子元素颜色被统一成一种 |
| 15 | 上下左右位置与坐标/视觉不一致 |
| 16 | Hero 与内容区重叠量靠猜 |
| 17 | 嵌套层级与 layer/Spec 不一致 |
| 18 | 多张卡片 padding 被写成同一个值 |
| 19 | 尺寸像「差不多」，未反推数据 |
| 20 | 间距靠 margin 链，未用坐标差 |
| 21 | 白字+白底、对比度风险 |
| 22 | 从别页复制了具体 CSS 数值 |
| 23 | 同行多色、富文本只写一个 color |
| 24 | 切图已带背景却再包一层或尺寸减半 |

# 图片输入降级指南

当用户输入是设计稿图片/截图（而非蓝湖 URL）时，蓝湖专属数据不可用，需要降级处理。

对应 `SKILL.md` v3 五步流程时：

| 主流程 | 图片输入时的含义 |
|--------|------------------|
| 第 1 步 准备数据 | 下文「第 1 步降级」— 技术栈检测 + 纯视觉提取，无 MCP |
| 第 2 步 内容校对 | 仍须输出元素勾选清单、CSS 对照表、交互清单；数值以视觉分析为主并标注 `⚠`；用户确认后写缓存（`SKILL.md §2.6`） |
| 第 3 步 生成代码 | 下文「第 3 步降级」 |
| 第 4 步 验证交付 | 下文「第 4 步降级」 |
| 第 5 步 第二轮修正 | 与蓝湖模式相同，先读缓存再定点改 |

---

## 总体原则

图片输入缺少结构化数据（无 layer tree、无 sketch_annotations、无 tokens），所有设计值都需要通过**视觉分析**提取。精度上限低于蓝湖输入，但仍需尽力还原。

### 图片质量要求

| 属性 | 最低要求 | 推荐 |
|------|---------|------|
| 分辨率 | ≥ 750px 宽（移动端）或 ≥ 1280px 宽（PC 端） | 1x 设计稿原始尺寸 |
| 格式 | PNG、JPG、WebP | PNG（保留细节） |
| 清晰度 | 文字可辨认，颜色无明显压缩伪影 | 无压缩原始截图 |
| 完整性 | 包含完整页面或目标区域 | 整页截图 + 局部放大补充 |

低于最低要求时，在输出开头标注 `⚠ 图片分辨率不足（建议 ≥ 750px 宽），颜色和尺寸提取精度可能受影响`。

---

## 第 1 步降级：数据准备（图片输入）

蓝湖 URL 可调用 MCP 获取结构化数据，图片只能用视觉分析。

### 1.5 图片输入流程（扩展）

| 属性 | 蓝湖数据源 | 图片替代方案 |
|------|-----------|------------|
| 页面背景色 | 画布最底层填充 | 视觉分析图片边缘/空白区域的背景色 |
| 页面尺寸 | 画布宽度 | 从图片尺寸推断设计稿宽度（如 750px → 750rpx 基准） |
| 内容最大宽度 | 画布约束 | 无，标注 `⚠ 无法从图片推断内容最大宽度` |

### 切图处理降级

图片输入时无法通过蓝湖 MCP 下载切图。替代方案：

1. **图标** — 标注 `⚠ 需要设计人员提供图标文件`，代码中用占位图或 icon font 代替
2. **背景图** — 标注 `⚠ 需要设计人员提供背景图`，代码中先用纯色/渐变兜底
3. **装饰图** — 同上，标注缺失

## 第 3 步降级：代码生成（图片输入）

### 2.1 页面全局属性（图片降级）

蓝湖有画布层级数据，图片没有。降级方案：

| 属性 | 蓝湖数据源 | 图片替代方案 |
|------|-----------|------------|
| 页面背景色 | 画布最底层填充 | 视觉分析图片边缘/空白区域的背景色 |
| 页面尺寸 | 画布宽度 | 从图片尺寸推断设计稿宽度（如 750px → 750rpx 基准） |
| 内容最大宽度 | 画布约束 | 无，标注 `⚠ 无法从图片推断内容最大宽度` |

### 2.2 按区域生成（图片降级）

蓝湖有 layer tree 可以遍历每个元素，图片只能靠视觉识别。

#### 降级策略

1. **从上到下、从左到右**扫描图片，按视觉区域划分
2. **每个可见元素逐一判断分类**（DOM文字 / DOM形状 / 切图），规则不变
3. 布局策略判断规则不变（流式 vs 重叠），但坐标数据降级为视觉估算

#### 图片输入的局限

- **无法获取被遮挡的元素**（如果有重叠元素，只能看到最上层的）
- **无法获取精确的坐标数据**（需要用视觉估算替代）
- **无法获取隐藏/折叠状态**（如果有展开/收起的状态，只能还原当前可见状态）

### 2.3 CSS 值提取（图片降级）

#### 颜色提取

| 信息 | 蓝湖数据源 | 图片替代方案 |
|------|-----------|------------|
| 精确色值 | tokens / fill | 视觉分析提取（标注 `⚠ 色值来自视觉分析，可能存在偏差`） |
| 渐变角度和色值 | gradient 数据 | 视觉分析提取大致渐变方向和颜色 |
| 文字颜色 | text color token | 视觉分析提取 |

**精度说明：** 视觉分析的色值可能有 ±5% 的偏差，标注来源为"图片视觉分析"。

#### 布局方向

| 信息 | 蓝湖数据源 | 图片替代方案 |
|------|-----------|------------|
| 元素上下左右 | x/y 坐标 | 视觉判断（相对更不可靠，需特别标注） |
| 重叠关系 | layer tree 层级 | 视觉判断 |

**关键：** 图片输入时，布局方向判断的可靠性降低。标注 `⚠ 布局方向基于视觉判断，请对照设计稿确认`。

#### 尺寸计算

| 需要计算的值 | 蓝湖数据源 | 图片替代方案 |
|------------|-----------|------------|
| hero 重叠量 | sketch_annotations Y 坐标 | 视觉估算（标注 `⚠ 重叠量基于视觉估算`） |
| 卡片内边距 | layer tree 外框/内容宽度 | 视觉估算 |
| 元素间距 | 绝对坐标差值 | 视觉估算 |
| 元素尺寸 | layer tree / slices.size | 视觉估算或图片测量工具 |

**单位换算：** 如果知道设计稿基准宽度（如 375px 或 750px），仍可进行换算。否则标注 `⚠ 单位换算基于推断的设计稿宽度`。

### 2.4 Layer Tree 结构还原（图片降级）

蓝湖有完整的 layer tree，图片没有。

1. **用视觉区域划分代替 layer tree** — 将图片从上到下划分为几个大区域
2. **区域内元素按视觉顺序排列** — 从左到右、从上到下
3. **重叠元素凭视觉判断** — 如果两个元素明显重叠（如图标叠加在进度条上），标注 `position: relative/absolute`
4. **标注** `⚠ 嵌套关系基于视觉判断`

## 第 4 步降级：验证（图片输入）

四项校验仍然适用，但验证数据源从蓝湖精确数据降级为视觉分析：

| 校验项 | 蓝湖模式 | 图片降级 |
|--------|---------|---------|
| 可见性 | tokens 色值精确对比 | 视觉分析色值对比（精度略低） |
| 跨页复制 | tokens/annotations 交叉验证 | 无精确数据，需用户确认 |
| 富文本 | 逐字核对颜色 | 同蓝湖模式，仍需逐字核对 |
| 切图尺寸 | slices.size 数据 | 无切图数据，标注 `⚠ 需人工确认尺寸` |

---

## 图片输入的特殊注意

1. **向用户说明精度差异** — 图片输入的还原精度天然低于蓝湖输入，需要在第一次输出时告知用户
2. **鼓励用户补充蓝湖链接** — 如果图片来源于蓝湖，尽量请用户提供蓝湖链接以获得更高精度
3. **提供验证手段** — 生成代码后，建议用户用浏览器打开并与设计图并排对比
